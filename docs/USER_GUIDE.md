# MeshKit 用户指南

[English](./en/USER_GUIDE.md) | 简体中文

本文档面向日常使用者，按任务说明 MeshKit 1.1.0 的基本操作。更短的入口页请看 [README](../README.md)。

## 启动方式

开发环境中可以在仓库根目录执行：

```bash
pnpm install
pnpm --filter core build
pnpm dev
```

也可以按模块单独启动：

```bash
pnpm dev:signaling
pnpm dev:web
pnpm dev:desktop
```

Docker 部署方式请看 [Docker 部署指南](./DOCKER_DEPLOYMENT.md)。Windows 和 macOS 安装包构建请看 [版本发布说明](./VERSION_RELEASE.md)。

## 文件传输

### 点对点发送

发送方：

1. 进入“文件传输”。
2. 选择点对点传输模式。
3. 选择文件，或把文件拖入页面。
4. 选择目标设备。
5. 点击发送，等待对方确认。

接收方：

1. 保持在“文件传输”页面。
2. 收到请求后查看文件列表。
3. 接受传输并选择需要保存的文件。
4. 等待文件传输完成。
5. 保存文件后点击“标记为已完成”。

使用提示：

- 传输期间双方尽量不要离开当前页面。
- 发送方可以取消发送。
- 如果对方一直看不到设备，先检查 signaling 服务、局域网和防火墙。

### 使用取件码

发送方：

1. 进入“文件传输”。
2. 选择生成取件码。
3. 添加要分享的文件。
4. 生成 6 位取件码。
5. 把取件码、邀请链接或二维码发给对方。

接收方：

1. 进入“文件传输”。
2. 选择输入取件码，或直接打开邀请链接。
3. 加入房间后保存需要的文件。
4. 完成后点击“标记为已完成”。

如果 RTC 状态异常，可以尝试点击“刷新 RTC”。如果发送方取消分享，接收方会收到提示。

## 便签墙

创建房间：

1. 进入“便签墙”。
2. 输入用户名。
3. 输入房间号，或使用随机房间号。
4. 按需设置房间密码和内容加密。
5. 创建并进入房间。

加入房间：

1. 输入用户名和房间号。
2. 如果房间启用了密码，输入正确密码。
3. 进入后即可协作编辑便签。

分享和管理：

- 可以通过邀请链接或二维码邀请其他成员。
- 最近房间支持快速返回。
- 页面会显示房主信息。
- 只有房主可以销毁房间。
- 房间销毁后，其他成员会收到提醒并退出。

## 加密聊天

创建或加入房间：

1. 进入“加密聊天”。
2. 输入用户名。
3. 输入房间号，或使用随机房间号。
4. 按需设置房间密码。
5. 进入房间后开始聊天。

分享和管理：

- 支持邀请链接、二维码和二维码下载。
- Desktop 可以导入这类邀请链接。
- 页面会显示房主信息。
- 只有房主可以销毁房间。
- 房间销毁后，其他成员会收到提醒并退出。

## Desktop 共享中心

Desktop 版适合希望用桌面程序启动本地共享入口的时候，当前支持 Windows 和 macOS。

1. 启动 MeshKit Desktop。
2. 在设置或共享入口中启动本地共享服务。
3. 把共享地址发给局域网内其他设备。
4. 如果收到 Web 端邀请链接，可以粘贴到 Desktop 中导入。
5. Desktop 会自动识别链接并跳转到对应功能页。

如果局域网设备无法访问 Desktop 共享地址，请检查主机 IP、端口和防火墙。macOS 上如果系统服务占用了 `7000`，Desktop 会自动换到后续可用端口，分享链接里会带上实际端口。

## 常见问题

### 看不到其他设备

- 确认双方在同一网络，或网络之间可以互相访问。
- 确认 signaling 服务已启动。
- 确认 WebSocket 端口和 PeerJS 端口没有被防火墙拦截。Desktop 端以共享中心显示的实际端口为准。
- 确认页面中的连接地址配置正确。

### 邀请链接打不开

- 确认链接没有被聊天软件截断。
- 如果是局域网地址，确认当前设备也在同一网络。
- 如果链接来自 Desktop，确认 Desktop 共享服务仍在运行。

### 文件传输中断

- 双方重新进入同一个页面后再试一次。
- 点击“刷新 RTC”重新建立连接。
- 检查浏览器是否阻止了下载或页面后台运行。

### 房间被销毁后还能使用同一个房间号吗

可以。销毁表示清空这次房间状态和通知成员退出，不代表这个房间号永久失效。之后再次使用同一个房间号，会创建新的房间。

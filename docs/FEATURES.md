# MeshKit 功能详解

本文档说明 MeshKit 1.1.0 当前可用的主要能力。想快速上手请先看 [使用说明](./使用说明.md)，想了解实现方式请看 [架构说明](./ARCHITECTURE.md)。

## 功能地图

| 模块 | 适合做什么 | 关键能力 |
| --- | --- | --- |
| 文件传输 | 临时发送文件、局域网互传、多文件队列 | 点对点传输、取件码、邀请链接、二维码、接收确认 |
| 便签墙 | 多人临时记录、协作整理信息 | Yjs 同步、房间密码、内容加密、房主销毁房间 |
| 加密聊天 | 临时私密沟通 | 房间密码、邀请链接、二维码、最近房间、房间销毁提醒 |
| Desktop | Windows/macOS 桌面端共享入口 | 内置信令服务、共享网页服务、导入邀请链接 |
| Docker | 局域网或服务器部署 | Web 静态站点、signaling 服务、健康检查 |

## 文件传输

### 点对点传输

点对点传输适合两台设备都在线、需要直接发送文件的时候。

- 发送方可以选择单个或多个文件，并选择在线目标设备。
- 接收方会先看到待接收文件列表，可以确认后再开始接收。
- 接收方可以逐个保存文件，避免自动写入本地文件系统。
- 接收方保存完成后，需要点击“标记为已完成”，让发送方知道这次传输已经结束。
- 发送方在等待确认、传输中、等待完成阶段都可以取消发送。

传输期间双方应尽量停留在当前页面。页面会在活跃传输时限制离开，减少中断和状态丢失。

### 取件码传输

取件码适合“我先创建一个取件房间，对方稍后加入领取”的用法。

- 发送方创建 6 位取件码。
- 支持复制取件码、邀请链接、展示二维码、下载二维码。
- 接收方可以输入取件码，也可以直接打开邀请链接加入。
- 接收方加入后逐个保存文件，完成后点击“标记为已完成”。
- 发送方取消发送后，接收方会收到明确提醒。
- 页面提供“刷新 RTC”入口，用于处理 WebRTC 建连异常。

### 连接和安全

文件内容通过 WebRTC DataChannel 在设备之间传输，signaling 服务只负责设备发现、房间状态和连接协商，不保存文件内容。

## 便签墙

便签墙用于多人协作记录，底层使用 Yjs 和 WebRTC 同步。

- 支持创建房间、加入房间、最近房间快速返回。
- 支持邀请链接、二维码和二维码下载。
- 支持房间密码校验，也支持便签内容加密配置。
- 页面显示房主信息，只有房主可以销毁房间。
- 房间销毁后，其他成员会收到提醒并退出当前房间。
- 移动端和窄屏布局已针对分享弹层、房间操作和便签操作做过适配。

便签墙更适合几个人临时协作。重要内容建议另外保存。

## 加密聊天

加密聊天用于临时房间沟通。

- 支持创建或加入房间。
- 支持房间密码校验。
- 支持最近房间记录。
- 支持邀请链接、二维码和二维码下载。
- 页面显示房主信息，只有房主可以销毁房间。
- 房间销毁后，其他成员会收到提醒并退出当前房间。

邀请链接可能包含房间号、连接参数或密码相关信息，请只分享给可信的人。

## Desktop 共享中心

Desktop 版本基于 Electron，当前支持 Windows 和 macOS，提供桌面端入口和本地共享能力。

- 可以启动内置信令服务和共享网页服务。
- 可以导入 Web 端生成的完整邀请链接。
- 导入后会自动识别目标功能，并跳转到取件码、便签墙或加密聊天页面。
- 桌面端图标、窗口标题和 Web 端品牌图标已统一为 MeshKit。
- macOS 上如果 AirPlay Receiver 等系统服务占用了默认端口，Desktop 会自动尝试后续可用端口。

局域网使用时，可能需要放行系统防火墙中的相关端口。

## 邀请链接与二维码

MeshKit 在文件传输、便签墙、加密聊天中使用统一的分享能力。

- 复制邀请链接。
- 展示二维码。
- 下载二维码图片。
- 支持系统分享能力的设备可以直接调用系统分享。
- Desktop 可以导入完整邀请链接。

如果需要跨设备分享，推荐使用二维码。如果需要跨应用转发，推荐复制完整邀请链接。

## 当前限制

- P2P 连接依赖双方网络环境，复杂 NAT 或严格防火墙下可能需要额外配置。
- 浏览器下载、剪贴板和系统分享权限受浏览器安全策略影响。
- 当前 signaling 服务不做账号体系和长期权限管理。
- 邀请链接默认给认识的人临时使用，不建议公开发布。

# MeshKit Feature Details

[简体中文](../FEATURES.md) | English

This document describes the main capabilities currently available in MeshKit `1.1.0`. For quick onboarding, start with [Quick Start](./QUICK_START.md). For implementation details, see [Architecture](./ARCHITECTURE.md).

## Feature Map

| Module | Best for | Key capabilities |
| --- | --- | --- |
| File Transfer | Temporary file sending, LAN transfer, multi-file queues | Peer-to-peer transfer, pickup codes, invite links, QR codes, receive confirmation |
| Notes Wall | Temporary multi-user notes and information organization | Yjs sync, room password, content encryption, owner-only room destruction |
| Encrypted Chat | Temporary private communication | Room password, invite link, QR code, recent rooms, room destruction notification |
| Desktop | Windows/macOS desktop sharing entry | Built-in signaling service, shared web service, invite link import |
| Docker | LAN or server deployment | Web static site, signaling service, health checks |

## File Transfer

### Peer-to-peer Transfer

Peer-to-peer transfer is suitable when both devices are online and files need to be sent directly.

<p align="center">
  <img src="../images/filetransfer.png" alt="File transfer" width="720">
</p>

- The sender can select one or more files and choose an online target device.
- The receiver sees the pending file list and can confirm before receiving.
- The receiver saves files one by one, avoiding automatic writes to the local filesystem.
- After saving files, the receiver should click Mark as Finished so the sender knows the transfer has ended.
- The sender can cancel while waiting for confirmation, transferring, or waiting for completion.

Both sides should stay on the current page during transfer. The page restricts leaving during active transfers to reduce interruptions and state loss.

### Pickup-code Transfer

Pickup codes are useful when the sender wants to create a temporary pickup room and let the other side join later.

<p align="center">
  <img src="../images/filetransfer2.png" alt="Pickup-code sending" width="720">
</p>

<p align="center">
  <img src="../images/filetransfer3.png" alt="Pickup-code receiving" width="720">
</p>

- The sender creates a 6-digit pickup code.
- Supports copying pickup codes, invite links, QR display, and QR download.
- The receiver can enter the pickup code or open an invite link directly.
- After joining, the receiver saves files one by one and clicks Mark as Finished.
- If the sender cancels, the receiver gets a clear notification.
- The page provides Refresh RTC for WebRTC connection issues.

### Connection and Security

File contents are transferred between devices through WebRTC DataChannel. Signaling only handles device discovery, room state, and connection negotiation. It does not store file contents.

## Notes Wall

Notes Wall is for collaborative note taking and lightweight boards, using Yjs and WebRTC for sync.

<p align="center">
  <img src="../images/noteswall.png" alt="Notes Wall" width="720">
</p>

- Supports room creation, room joining, and quick return through recent rooms.
- Supports invite links, QR codes, and QR downloads.
- Supports room password verification and optional content encryption.
- Shows owner information, and only the owner can destroy the room.
- Other members are notified and leave after room destruction.
- Mobile and narrow layouts are optimized for share dialogs, room actions, and note actions.

Notes Wall is best for temporary collaboration between a few people. Important content should be saved separately.

## Encrypted Chat

Encrypted Chat is for temporary room-based conversation.

<p align="center">
  <img src="../images/chat.png" alt="Encrypted chat" width="720">
</p>

- Supports creating or joining rooms.
- Supports room password verification.
- Supports recent room records.
- Supports invite links, QR codes, and QR downloads.
- Shows owner information, and only the owner can destroy the room.
- Other members are notified and leave after room destruction.

Invite links may contain room IDs, connection parameters, or password-related information. Share only with trusted people.

## Desktop Share Hub

The Desktop version is based on Electron, currently supports Windows and macOS, and provides a local sharing entry.

- Can start the built-in signaling service and shared web service.
- Can import full invite links generated by the Web app.
- Automatically detects the target feature and jumps to pickup code, Notes Wall, or Encrypted Chat.
- Desktop icons, window title, and Web brand icons are unified as MeshKit.
- On macOS, if AirPlay Receiver or other system services occupy default ports, Desktop automatically tries later available ports.

LAN usage may require allowing related ports in the system firewall.

## Invite Links and QR Codes

MeshKit uses a unified sharing capability across File Transfer, Notes Wall, and Encrypted Chat.

- Copy invite links.
- Show QR codes.
- Download QR images.
- Use system share on devices that support it.
- Import full invite links in Desktop.

Use QR codes for cross-device sharing. Use full invite links for forwarding across apps.

## Current Limitations

- P2P connections depend on both network environments. Complex NAT or strict firewall may require additional configuration.
- Browser downloads, clipboard, and system share permissions are affected by browser security policies.
- The current signaling service does not provide account systems or long-term permission management.
- Invite links are intended for temporary sharing with people you know and should not be published publicly.

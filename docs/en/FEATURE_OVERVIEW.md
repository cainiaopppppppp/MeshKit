# MeshKit Feature Overview

[简体中文](../功能介绍.md) | English

This document describes the current MeshKit `1.1.0` feature set.

## 1. File Transfer

MeshKit file transfer has two modes: peer-to-peer transfer and pickup-code transfer.

<p align="center">
  <img src="../images/filetransfer.png" alt="File transfer" width="720">
</p>

### Peer-to-peer Transfer

- Select a target device from the online device list.
- Supports single files and multi-file queues.
- The receiver sees a receive request and can accept or reject it.
- During multi-file transfer, the receiver can select part of the files.
- After receiving, the receiver clicks Mark as Finished so the sender can exit the waiting state.
- The sender can cancel while waiting for confirmation, transferring, or waiting for receiver completion.

### Pickup-code Transfer

<p align="center">
  <img src="../images/filetransfer2.png" alt="Pickup-code sending" width="720">
</p>

<p align="center">
  <img src="../images/filetransfer3.png" alt="Pickup-code receiving" width="720">
</p>

- The sender selects files and generates a 6-digit pickup code.
- The receiver can enter the pickup code or join through an invite link or QR code.
- Files can be saved one by one.
- RTC connection can be refreshed.
- If the owner cancels sending, the receiver is notified and leaves the room.

### Transfer Protection

- Page switching is limited during active transfer to reduce accidental interruption.
- File data is transferred through WebRTC DataChannel.
- Signaling only handles device discovery, room state, and connection coordination. It does not store file contents.

## 2. Notes Wall

Notes Wall is for temporary multi-user collaboration and lightweight boards.

<p align="center">
  <img src="../images/noteswall.png" alt="Notes Wall" width="720">
</p>

- Real-time sync based on Yjs and WebRTC.
- Supports room passwords and content encryption settings.
- Supports recent rooms for quick return after refresh.
- Supports invite links, QR codes, and QR downloads.
- Shows owner information.
- Only the owner can destroy a room.
- Other members are notified and leave after room destruction.
- Mobile layout supports fixed top bar, bottom status bar, zoom controls, and long-press dragging.

<p align="center">
  <img src="../images/noteswall2.png" alt="Notes Wall room and sharing" width="720">
</p>

## 3. Encrypted Chat

Encrypted Chat is for temporary private discussion.

<p align="center">
  <img src="../images/chat.png" alt="Encrypted chat" width="720">
</p>

- Supports creating or joining rooms.
- Supports room password verification.
- Supports recent rooms and quick return.
- Supports invite links, QR codes, and QR downloads.
- Shows owner information.
- Only the owner can destroy a room.
- Other members are notified and leave after destruction.
- Supports message self-destruct time settings.

  <img src="../images/chat2.png" alt="Encrypted chat room" width="720">

## 4. Desktop Share Hub

The Desktop version provides local sharing capability in addition to UI and currently supports Windows and macOS.

<p align="center">
  <img src="../images/settings.png" alt="Desktop Share Hub settings" width="720">
</p>

- Automatically hosts the built-in signaling service.
- Automatically hosts the browser share page.
- Supports copying invite links.
- Supports showing QR codes.
- Supports importing full invite links generated by Web or Desktop.
- Automatically saves connection settings and jumps to the matching page after import.
- On macOS, if a system service occupies the default port, Desktop switches to later available ports automatically.

## 5. Invite Links and QR Codes

Invite links reduce the friction of LAN usage.

- The Web top bar can copy the current shareable link.
- File Transfer, Notes Wall, and Encrypted Chat can generate room-specific links.
- Share dialogs show QR codes and support QR image downloads.
- Desktop can read full invite links and auto-configure host, ports, and target feature.

Note: invite links may contain connection parameters, pickup codes, room IDs, or required password-related information. Share only with trusted people.
